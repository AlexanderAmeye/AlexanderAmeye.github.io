Introduction

This page is all about the Scriptable Rendering Pipeline in Unity3D. The goal of this page is to provide a general insight into the Unity3D rendering pipeline and provide a guide on how to customize it.


Rendering Requirements

For this particular rendering style I want the following key features.

>

When rendering in a Tintin-style, we generally want the things in the list below. However, for specific cases this might change and we want to be able to override them if needed.


  • Flat colors, no gradients
  • A single-width outline around all objects.
  • Only metallic or shiny objects are influenced by light.
  • Generally no shadows.

Edge Detection

Using an edge detection shader, we render an outline over several objects in the scene. This edge detection is added to the rendering pipeline as a custom pass in the form of a Renderer Feature.



Edge detection is based on 3 paramaters. Depth, Normals and Color. Depth + Normals are used for geometry and Color for lights and shadows. The edge detection pass occurs after transparent objects have been rendered so that outlines are also renderer around our fake volumetric lights.


Spotlights

For this particular rendering style, we want spotlights to have a hard outline. In some cases we also want the spotlight to have a volumetric effect. To get a hard outline, we need to customize the AngleAttenuation function in the Lighting.hlsl file that comes with the rendering pipeline.
